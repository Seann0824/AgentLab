# CLI UX 优化计划

## 目标
全面优化命令行用户体验，打造开发者友好的 CLI 交互体验。

## 关键问题识别
当前 CLI 存在的问题：
1. 提示符仅一个 `>` 字符，无上下文信息
2. 工具执行时只有静态 "⏳ 正在执行..." 文字，无动态反馈
3. 输出格式简单，缺乏视觉层次
4. 无欢迎信息/启动状态
5. 会话列表为纯文本，无表格格式
6. 流式输出直接打印，无格式化
7. 整体缺少"产品级"质感

## 执行步骤

- [x] 步骤 1: 创建 `cli/output.rs` 样式工具模块（颜色、图标、边框、表格、状态栏等）
- [x] 步骤 2: 优化用户提示符（带颜色、上下文信息、工作目录）→ 集成到 agent.rs
- [x] 步骤 3: 添加启动欢迎横幅（版本号、工作目录、命令提示）→ 集成到 agent.rs
- [x] 步骤 4: 添加加载动画/进度指示器（工具执行时）→ 集成到 agent.rs
- [x] 步骤 5: 优化工具调用和结果展示（格式化表格、状态徽章）→ 集成到 agent.rs
- [ ] 步骤 6: 优化会话管理展示（表格格式、更清晰的操作反馈）
- [ ] 步骤 7: 优化流式输出格式（更好的思考/文本区分）
- [ ] 步骤 8: 添加统一的成功/错误/警告视觉层次
- [ ] 步骤 9: 优化 Token 使用状态展示
- [ ] 步骤 10: cargo check 验证 + 自我验证
