# Agent 能力路线图 — 持久化记忆与总结

> 原文拆分自 `../agent-capability-roadmap.md`。

## 7. 阶段四：持久化与记忆（P3）

> **目标**: Agent 拥有跨会话的长期记忆
> **预期效果**: 多次对话之间能记住项目上下文、用户偏好
> **估算**: 3-5 天

### 7.1 记忆层级

```
┌──────────────────────────────────────────┐
│          工作记忆 (Working Memory)          │
│    当前任务状态、临时变量、步骤进度          │
│    载体: PLAN.md, AGENDA.md                │
├──────────────────────────────────────────┤
│          情景记忆 (Episodic Memory)         │
│    过去任务的经验、成功模式、错误教训        │
│    载体: MEMORY.md, 项目根目录的 .agent/   │
├──────────────────────────────────────────┤
│          语义记忆 (Semantic Memory)         │
│    项目结构知识、技术栈、约定规范            │
│    载体: .agent/knowledge.md               │
└──────────────────────────────────────────┘
```

### 7.2 存储结构

```
项目根目录/
├── .agent/                  # Agent 记忆目录
│   ├── memory.md            # 长期记忆（经验、决策、教训）
│   ├── knowledge.md         # 项目知识（结构、约定、技术选型）
│   └── sessions/            # 会话日志（可选）
│       └── 2025-01-15.md
├── PLAN.md                  # 当前执行计划
└── AGENDA.md                # 当前议程
```

### 7.3 关键设计：记忆检索

当 ContextManager 触发压缩时，将摘要内容同时写入 `.agent/memory.md`：

```
【注入时机】
ContextManager 压缩早期对话时
→ 同时将摘要追加到 .agent/memory.md
→ 下次会话启动时加载 memory.md 作为系统提示词的一部分
```

## 8. 总结

### 优先级排序

| 优先级 | 阶段 | 内容 | 预估工作量 | 效果 |
|--------|------|------|-----------|------|
| 🔴 **P0** | 一 | 结构化执行框架 | 系统提示词 + 0.5天编码 | 立即提升复杂任务完成率 |
| 🟡 **P1** | 二 | 自我进化能力 | 2-3天 | Agent 可自主扩展 |
| 🟢 **P2** | 三 | 多 Agent 协作 | 3-5天 | 专业分工处理复杂任务 |
| 🔵 **P3** | 四 | 持久化与记忆 | 3-5天 | 跨会话连续工作 |
