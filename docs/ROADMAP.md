## 🔴 Phase 2: 自我进化（待开始）

> **目标**: Agent 能够自主修改自身代码来获得新能力，实现「自我进化」  
> **进度**: 15% — 已有增量编辑工具，但缺少系统化的自我修改工作流

### 2.1 现状

| 能力 | 状态 | 说明 |
|------|------|------|
| 增量编辑工具 | ✅ | edit 工具支持 search/replace/insert/delete/append |
| 编译验证 | ✅ | 系统提示词引导修改后 cargo check |
| `app/` 用户代码目录 | ✅ | 框架与用户代码分离 |
| **新工具代码生成** | ❌ | 没有模板化的「生成新工具」工作流 |
| **自我修改工作流** | ❌ | 没有系统化的「分析→修改→验证→注册」流程 |
| **变更回滚** | ❌ | 没有版本管理或快照回滚机制 |

### 2.2 计划

#### Step 1: 新工具脚手架生成（P0）
```
目标: Agent 可以通过描述自动生成一个新工具的模板代码
- 实现 `generate_tool` 工具（或系统提示词引导 + 模板）
- 输入: 工具名 + 参数描述 + 功能描述
- 输出: 完整的 Rust 源文件 + 自动注册到 ToolManager
- 验证: cargo check 通过
```

#### Step 2: 自我修改安全机制（P1）
```
目标: 确保 Agent 修改自身代码时的安全性
- 自动备份修改前的文件
- 编译失败时自动回滚
- 关键文件（agent.rs, tools/mod.rs）的修改权限控制
```

#### Step 3: 能力自检与建议（P2）
```
目标: Agent 能发现自己缺少什么能力，并主动提议添加
- 任务分析 → 发现缺少必要工具 → 生成工具 → 注册 → 继续执行
```
